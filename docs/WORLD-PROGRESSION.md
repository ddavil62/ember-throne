## 1. 월드맵 구조

### 1.1 월드맵 비주얼 컨셉

- **양피지/고지도(Old Parchment Map)** 스타일의 월드맵.
- 대륙 에셀란드(Aetheland)를 위에서 내려다보는 구도.
- 5개 지역을 점선 도로로 연결하고, 각 지역에 거점 아이콘(건물/숲/산 등)을 배치한다.
- 현재 진행 가능한 거점만 활성화(밝은 아이콘 + 금빛 테두리). 나머지는 어둡게(그레이아웃 + 잠금 아이콘).
- 이미 클리어한 거점은 체크 마크 표시.
- 스토리 진행에 따라 맵의 상태가 변한다:
  - 결계석이 회수된 지역은 잿빛 안개가 서서히 퍼지는 연출.
  - 3막 결계석 순례 시 각 지역을 재방문하면 잿빛 안개가 걷히는 연출.
- 아트 키워드: `parchment world map, fantasy cartography, old ink drawing, golden highlights, fog of war, medieval map style`

### 1.3 진행 잠금 규칙

1. **순차 해금**: 스토리 씬 순서에 따라 거점이 해금된다. 이전 스토리 거점을 클리어해야 다음 거점에 접근 가능하다.
2. **이전 막 재방문**: 이전 막에서 해금된 거점은 언제든 재방문 가능하다. 상점 이용, 유랑 전투가 가능하다.
3. **3막 결계석 순례**: 3-10 ~ 3-12에서 이르헨, 하르벤, 실바렌, 벨마르, 크로우펠을 순서대로 재방문해야 한다. 스토리 이벤트와 전투가 각 지역에서 발생한다.
4. **4막 왕도 잠금**: 4막에 진입하면 왕도 외부 거점으로 돌아갈 수 없다 (포위전 상태). 상점은 이렌의 지하 조직을 통해 제한적으로 이용 가능하다.
5. **유랑 전투 해금**: 해당 지역의 스토리 전투를 최소 1회 클리어한 후 해금된다.
6. **최종전 진입 경고**: 4-7(최종 전투) 진입 전 "이후 돌아올 수 없습니다" 경고를 표시한다.

## 2. 세이브/로드 시스템

### 2.1 세이브 슬롯

| 유형 | 슬롯 수 | 타이밍 | 비고 |
|------|---------|--------|------|
| 수동 세이브 | 3슬롯 | 월드맵, 전투 준비 화면에서 가능 | 플레이어가 직접 저장 |
| 자동 세이브 | 1슬롯 | 전투 시작 전, 스토리 이벤트 시작 전 자동 저장 | 덮어쓰기 방식 |
| 전투 중 세이브 | 불가 | - | 전투는 중단 없이 진행 |

- 세이브 슬롯 UI에는 다음 정보를 표시:
  - 현재 스토리 위치 (예: "제2막 - 실바렌 방어전")
  - 플레이 시간
  - 카엘 레벨
  - 파티 인원 수
  - 세이브 시각
  - 스크린샷 미리보기 (월드맵 or 전투 준비 화면 캡처)

### 2.2 세이브 데이터 구조

```
SaveData {
    // 메타 정보
    save_version: int           // 세이브 포맷 버전
    play_time: float            // 총 플레이 시간 (초)
    save_timestamp: string      // 저장 시각 (ISO 8601)
    difficulty: string          // "normal" | "hard"

    // 스토리 진행
    current_act: int            // 현재 막 (1~4)
    current_scene: string       // 현재 씬 번호 (예: "2-4")
    completed_scenes: string[]  // 완료한 씬 목록
    unlocked_nodes: string[]    // 해금된 월드맵 거점 ID 목록

    // 캐릭터 상태
    characters: {
        [character_id]: {
            level: int
            exp: int
            hp_current: int
            mp_current: int
            stats: {
                hp_max, mp_max, atk, def, matk, mdef, spd, mov
            }
            class_name: string      // 현재 클래스
            is_promoted: bool       // 전직 여부
            equipped_weapon: string
            equipped_armor: string
            equipped_accessory: string
            active_skills: string[] // 장착 액티브 스킬 (최대 6개)
            learned_skills: string[] // 습득한 전체 스킬
        }
    }

    // 유대 시스템
    bonds: {
        [bond_pair_id]: {
            level: int              // 0~3
            adjacent_battle_count: int // 인접 배치 전투 횟수 (Lv3 조건용)
        }
    }

    // 인벤토리
    inventory: {
        consumables: { [item_id]: int }  // 소비 아이템: 수량
        equipment: string[]               // 미장착 장비 목록
        key_items: string[]               // 핵심 아이템 목록
        materials: { [item_id]: int }     // 소재: 수량
    }

    // 경제
    gold: int

    // 도감
    codex: {
        enemies_encountered: string[]     // 조우한 적 ID
        items_obtained: string[]          // 획득한 적 있는 아이템 ID
        story_scenes_viewed: string[]     // 감상한 스토리 씬
        cg_unlocked: string[]             // 해금된 CG ID
    }

    // 전투 기록
    battle_records: {
        total_battles: int
        total_victories: int
        total_defeats: int
        wandering_battles_completed: string[] // 클리어한 유랑 전투 ID
    }

    // 설정 (세이브별 독립)
    settings: {
        bgm_volume: float
        se_volume: float
        text_speed: string          // "slow" | "normal" | "fast" | "instant"
        auto_advance: bool
        battle_animation: string    // "full" | "short" | "skip"
    }
}
```

### 2.3 세이브 관련 규칙

- **전투 패배 시**: 해당 전투 시작 전 자동 세이브로 자동 복귀. 경험치/아이템 유지 안 됨.
- **데이터 무결성**: 세이브 파일에 체크섬을 포함하여 변조를 감지한다.
- **Steam Cloud**: GodotSteam 플러그인을 통해 클라우드 세이브를 지원한다 (예정).
- **뉴 게임+**: 1회 클리어 후 해금. 캐릭터 레벨, 장비를 이월하고 스토리를 다시 시작한다. 다른 엔딩 경험 용도.

## 3. 유대 시스템 (Bond System)

### 3.1 개요

특정 캐릭터 쌍 사이에 "유대 레벨(Bond Level)"이 존재한다. 유대 레벨이 오르면 전투 중 인접(4방향) 배치 시 보정 효과가 강화된다. 유대는 스토리 서사와 연동되며, 플레이어의 전투 편성에 전략적 깊이를 더한다.

### 3.2 유대 쌍 목록

| 쌍 | 유대 유형 | 최대 레벨 | Lv1 보정 | Lv2 보정 | Lv3 보정 | 서사 근거 |
|----|-----------|-----------|----------|----------|----------|-----------|
| 카엘 ↔ 세리아 | 연인 | Lv3 | 쌍방 ATK+3%, DEF+3% | 쌍방 ATK+7%, DEF+7%, 치명타율+5% | 쌍방 ATK+10%, DEF+10%, 치명타율+10% | 1막에서 만나 여정을 함께하며 사랑이 깊어진다 |
| 리넨 ↔ 나엘 | 연인 | Lv3 | 쌍방 MATK+3%, MDEF+3% | 쌍방 MATK+7%, MDEF+7%, HP재생 3%/턴 | 쌍방 MATK+10%, MDEF+10%, HP재생 5%/턴 | 실바렌에서 시작된 사랑, 리넨의 기억 여정을 함께 |
| 볼드 ↔ 에리스(NPC) | 부부 | Lv2 | 볼드 DEF+5% | 볼드 DEF+15% (에리스 생존 시) | - | 인질로 잡힌 아내를 위한 의지. 에리스 사망 후 Lv2 효과가 "에리스의 방패" 패시브로 전환 |
| 카엘 ↔ 로크 | 전우 | Lv3 | 쌍방 명중+5%, 회피+3% | 쌍방 명중+10%, 회피+7% | 쌍방 명중+15%, 회피+10% | 서로의 상실을 이해하는 전우. 실바렌에서의 공동 전선 |
| 세리아 ↔ 엘미라 | 사제 | Lv2 | 쌍방 치유효과+10% | 쌍방 치유효과+20% | - | 이단 선고의 배경, 재회와 화해, 함께하는 의식 |
| 그리드 ↔ 로크 | 악우 | Lv2 | 쌍방 SPD+5% | 쌍방 SPD+15% | - | 성격은 다르지만 통하는 두 사람. 벨마르~실바렌 여정의 동료 |
| 드라나 ↔ 세리아 | 학우 | Lv2 | 쌍방 MATK+5% | 쌍방 MATK+10% | - | 마법과 신앙이라는 다른 길을 걷지만 지식을 나누는 동료 |
| 카엘 ↔ 헤이즐 | 대칭 | Lv2 | 쌍방 치명타율+5% | 쌍방 치명타율+15% | - | "명령을 따른 자 vs 명령을 거부한 자" -- 거울 같은 두 군인 |
| 이렌 ↔ 볼드 | 동료 | Lv2 | 쌍방 DEF+5% | 쌍방 DEF+10% | - | 모르간에게 가족을 잃은 공통의 상처. 에필로그에서 고아원을 함께 운영 |
| 리넨 ↔ 드라나 | 학술 | Lv2 | 쌍방 MATK+3%, MDEF+3% | 쌍방 MATK+7%, MDEF+7% | - | 드라나의 결계석 지식이 리넨의 정체 해명에 기여 |
| 그리드 ↔ 이렌 | 암거래 | Lv2 | 쌍방 명중+5% | 쌍방 명중+10%, SPD+5% | - | 밀수업자와 지하 조직 수장. 뒷세계의 정보 네트워크를 공유 |

### 3.3 유대 레벨업 조건

| 전환 | 조건 유형 | 상세 |
|------|-----------|------|
| Lv0 -> Lv1 | 자동 (합류 시) | 해당 캐릭터 쌍이 모두 파티에 합류하면 자동으로 Lv1 |
| Lv1 -> Lv2 | 스토리 이벤트 (자동) | 특정 스토리 씬 트리거 시 자동 레벨업 |

### 3.4 유대 효과 적용 규칙

- 인접(4방향, 상하좌우)에 유대 쌍 상대가 있을 때만 보정 적용.
- 대각선은 인접으로 간주하지 않는다.
- 같은 유닛이 여러 유대 쌍에 속할 수 있으며, 보정은 중첩된다 (예: 카엘이 세리아 + 로크에 인접하면 양쪽 보정 모두 적용).
- 전투불능 유닛의 유대 보정은 적용되지 않는다.
- 유대 보정은 전투 전 편성 화면에서 "인접 배치 시 효과" 미리보기로 확인할 수 있다.

### 5.3 사운드 규칙

1. **BGM 전환**: 크로스페이드(1.5초) 방식. 씬 전환 시 자연스러운 연결.
2. **전투 BGM**: 전투 시작 시 BGM-07/08/09로 전환. 전투 종료 후 이전 BGM으로 복귀.
3. **이벤트 BGM 우선**: 스토리 이벤트 BGM은 월드맵/전투 BGM보다 우선한다.
4. **볼륨 계층**: BGM(기본 100%) + SE(기본 100%) + 환경음(기본 50%). 각각 독립 볼륨 슬라이더.
5. **동시 재생 제한**: SE는 최대 8채널 동시 재생. 우선순위 A > B. 채널 초과 시 B등급부터 드롭.
6. **무음 구간**: 극적 순간(에리스의 죽음, 렌도르의 속죄 등)에서 BGM을 페이드아웃하고 SE만 남겨 감정 효과를 극대화한다.
